Repo detail: how to navigate?

Currently, tapping a repo opens an expansion tile inline. The new design needs a full settings page per repo. How should the user get there?

A

Tap repo name → full page

The repo list becomes a simple list of cards. Tapping anywhere on the card navigates to /repos/:repo — a dedicated full-screen settings page. The toggle switch stays inline for quick enable/disable without navigating.

Pros

  • Room for all settings (issue tracking, AI, labels, local dir, PR metadata)
  • Clean separation — list is for browsing, detail page is for editing
  • Matches PRDetailScreen / IssueDetailScreen pattern

Cons

  • Extra navigation step to change a single setting
B

Keep expansion tile + "Full settings" button

The expansion tile stays for quick overrides (AI primary/fallback). A "Full settings" button inside the tile navigates to the full page for advanced config (issue tracking labels, PR metadata).

Pros

  • Quick access to common settings inline
  • Advanced settings don't clutter the list

Cons

  • Two places to configure the same repo — confusing
  • Expansion tile already cramped with current fields
C

Side panel (master-detail)

Repo list on the left, settings panel on the right. Clicking a repo shows its full config in the right panel without navigating away.

Pros

  • No navigation — everything visible at once
  • Fast switching between repos

Cons

  • Needs wide screen — desktop-only pattern
  • Different from all other screens in the app (PRs, Issues use full-page detail)
